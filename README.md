# Cyber Security Case Study – Safe Drive Automobiles (SDA)
## My Approach

In this project, I focused on identifying practical vulnerabilities and linking them to real world risks. I analysed each issue based on its impact and prioritised solutions that align with industry best practices.

In this project, I analysed a simulated real-world cyber attack on a medium-sized automobile company.
## Scenario
The company was targeted by a phishing-based attack that resulted in a triple extortion incident. Sensitive customer data was exposed, including financial and personal information.

## Key Issues Identified

- Unpatched operating systems
- Weak password policies
- Broken cryptography (MD4, SSL 2.0)
- Insecure remote access (RDP)
- Unsafe data disposal
- Lack of physical security controls

## Risk Analysis
Each issue was analysed based on its impact on:
- Confidentiality
- Integrity
- Availability (CIA triad)

## Proposed Solutions

- Strong password policies + MFA
- Upgrade to modern encryption (TLS 1.3, AES)
- Secure remote access using VPN + restrictions
- Implement proper patch management
- Improve physical security and redundancy

## Key Learning

This project helped me understand:
- How real-world cyber attacks occur
- How to identify vulnerabilities in systems
- The importance of layered security controls
- Practical application of the CIA triad

This project was completed as part of my cyber security studies and reflects my approach to analysing security risks.
